## 一、环节概述

### 1.1 环节定位

```
designing 阶段是 clawdevflow 流程的第一个阶段
输入：REQUIREMENTS.md（需求说明）
输出：PRD.md（产品需求文档）+ TRD.md（技术设计文档）
审阅：ReviewDesignAgent v3.1.0
```

### 1.2 环节流程

```
REQUIREMENTS.md
    ↓
Workflow Orchestrator 调用 executeStage('designing')
    ↓
Stage Executor 准备输入
    ↓
AI Tool Adapter 调用 OpenCode
    ↓
OpenCode 执行 designing skill
    - 阅读 REQUIREMENTS.md
    - 生成 PRD.md（产品需求文档）
    - 生成 TRD.md（技术设计文档）
    ↓
检查输出文件（PRD.md + TRD.md）
    ↓
ReviewDesignAgent v3.1.0 审阅
    - D1 需求覆盖率（含版本对齐 + 可定位映射）
    - D2 文档完整性
    - D3 无模糊词
    - D4 技术选型合理
    - D5 向后兼容
    - D6 异常处理
    - D7 验收标准可测试性
    ↓
用户审阅（最终确认）
```

### 5.3 异常处理完整 ✅

```markdown
### 10.2 异常处理

#### 10.2.1 正常流程
- 阶段执行成功 → 进入下一阶段
- 审阅通过 → 继续流程

#### 10.2.2 失败处理
- 阶段执行失败 → 记录错误 → 终止流程
- 审阅驳回 → 重新执行当前阶段（最多 3 次）

#### 10.2.3 边界情况
- 超时处理：timeoutSeconds 配置
- 并发限制：单流程单实例
- 数据异常：JSON 解析失败回滚

#### 10.2.4 重试机制
- 网络请求失败：重试 3 次，指数退避
- 文件写入失败：重试 3 次，检查磁盘空间

#### 10.2.5 降级方案
- AI 工具不可用：降级为手动模式
- 文件系统不可用：使用内存临时存储
- 网络不可用：使用本地缓存配置

#### 10.2.6 监控告警
- 流程执行时间 > 30 分钟 → 告警
- 连续失败 3 次 → 告警
- 磁盘空间 < 10% → 告警
- 内存使用 > 80% → 告警
```
