# 文档审查清单

编码前审查用户提供的文档。质疑问题，而不是照单全收。

## 审查清单

```
文档审查：
- [ ] 需求有效性
- [ ] 技术可行性
- [ ] 方案设计
- [ ] API 设计
- [ ] 依赖合理性
- [ ] 与现有代码的一致性
```

## 审查维度

| 维度         | 检查内容             |
|------------|------------------|
| **需求有效性**  | 是否解决真实问题？是否过度设计？ |
| **技术可行性**  | 可实现吗？有更好的替代方案吗？  |
| **方案设计**   | 方案准确吗？是否有模糊或冗余？  |
| **API 设计** | 遵循约定吗？参数合理吗？     |
| **依赖合理性**  | 必要吗？有更轻量的替代方案吗？  |
| **一致性**    | 是否匹配现有架构/命名/风格？  |

## 问题报告格式

发现问题时：

```markdown
## 文档审查

**[问题类型]**：[位置] - [具体内容]
**问题**：[描述]
**影响**：[不修复会怎样]
**建议**：[改进方案]

编码前修复？
```

## 无问题

简单声明："文档审查通过"，然后继续实施。
