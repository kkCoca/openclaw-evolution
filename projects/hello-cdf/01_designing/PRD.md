## 1. 产品概述

hello-cdf 是一个简单的命令行问候工具，用于验证 CDF 研发流程的正确性。

**目标用户**:
- 命令行开发者
- 需要快速测试 CDF 流程的用户

**核心价值**:
- 一次提交，多仓同步
- 完整的研发流程验证

---

## 2. 功能需求

### 2.1 默认问候功能 **[REQ-001]**

**需求描述**: 运行 `hello-cdf` 输出 "Hello, CDF!"

**功能设计**:
- 用户运行 `hello-cdf` 命令
- 程序检测无参数
- 输出默认问候语 "Hello, CDF!"

**验收标准**:
```gherkin
Given 用户在终端
When 运行 `hello-cdf` 不提供参数
Then 输出 "Hello, CDF!"
```

---

### 2.2 自定义名字功能 **[REQ-002]**

**需求描述**: 运行 `hello-cdf --name <名字>` 输出 "Hello, <名字>!"

**功能设计**:
- 用户运行 `hello-cdf --name Alice`
- 程序解析 `--name` 参数
- 输出个性化问候语 "Hello, Alice!"

**验收标准**:
```gherkin
Given 用户提供名字参数
When 运行 `hello-cdf --name Alice`
Then 输出 "Hello, Alice!"
```

---

### 2.3 帮助信息功能 **[REQ-003]**

**需求描述**: 运行 `hello-cdf --help` 显示使用说明

**功能设计**:
- 用户运行 `hello-cdf --help`
- 程序检测 `--help` 参数
- 显示使用说明和参数列表
- 退出程序（不执行其他逻辑）

**验收标准**:
```gherkin
Given 用户需要帮助
When 运行 `hello-cdf --help`
Then 显示使用说明和参数列表
```

---

### 2.4 版本信息功能 **[REQ-004]**

**需求描述**: 运行 `hello-cdf --version` 显示版本号

**功能设计**:
- 用户运行 `hello-cdf --version`
- 程序检测 `--version` 参数
- 显示 "hello-cdf v1.0.0"
- 退出程序

**验收标准**:
```gherkin
Given 用户查询版本
When 运行 `hello-cdf --version`
Then 显示 "hello-cdf v1.0.0"
```

## 3. 非功能需求

| 需求 | 说明 | 验收标准 |
|------|------|---------|
| **性能** | 启动时间 < 100ms | 从运行命令到输出结果 < 100ms |
| **代码简洁** | 不超过 50 行 | 源代码行数 ≤ 50 |
| **无外部依赖** | 仅使用 Node.js 标准库 | package.json 无 dependencies |
| **跨平台** | 支持 Linux/macOS/Windows | 三个平台都能正常运行 |
