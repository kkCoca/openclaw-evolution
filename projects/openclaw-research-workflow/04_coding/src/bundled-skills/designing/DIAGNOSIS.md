# 审查模式诊断

审查现有文档时使用此清单。

## 诊断清单

```
审查诊断：
- [ ] 模板格式
- [ ] 覆盖完整性
- [ ] 模糊描述
- [ ] 需求文档 ↔ 技术文档 一致性
- [ ] 类型规格
- [ ] 技术选型有效性
- [ ] 禁止在trd中写实现
```

## 覆盖完整性

检查必需章节是否存在：

**需求文档**：

- [ ] 目标用户已定义
- [ ] 痛点已识别
- [ ] 功能带验收标准
- [ ] 流程包含失败路径
- [ ] 异常处理
- [ ] 范围边界

**技术文档**：

- [ ] 技术选型带理由
- [ ] 类型集中定义
- [ ] 数据模型完整
- [ ] 模块职责清晰
- [ ] 错误策略已定义

## 模糊描述检测

搜索禁用术语：

- "适当的" / "合适的" / "恰当的"
- "一些" / "某些" / "各种"
- "待定" / "TODO" / "以后"
- "等等" / "诸如此类"
- "可能" / "也许" / "或许"

## 一致性检查

| 需求文档章节 | 必须匹配 | 技术文档章节 |
|--------|------|--------|
| 功能     | →    | 模块     |
| 数据需求   | →    | 实体字段   |
| 异常     | →    | 错误策略   |
| 术语     | =    | 术语     |

## 输出格式

```markdown
## 诊断结果

| 问题 | 位置 | 严重程度 | 建议 |
|------|------|----------|------|

## 修复优先级

1. [先修复关键问题]
2. ...
```

用户确认后，逐一修复问题，尽量提供工具让用户快速选择而不是粗暴的要求用户输入回答。
