## 涉及模块

- `src/index.ts`：Skill 入口与 `smartSearch` 对外导出。
- `src/fallback-manager.ts`：Gemini -> DDG 状态流转、重试、超时与最终错误出口。
- `src/error-classifier.ts`：错误归类、降级触发条件与不可降级错误判断。
- `src/ddg-adapter.ts`：DuckDuckGo Provider 适配与响应结构统一。
- `src/types.ts`：搜索输入、输出、错误、配置等核心类型。
- `tests/`：fallback 行为、分类逻辑、边界失败场景测试。
- `README.md`：集成说明、配置示例、使用方式、实验记录口径。

## 交付节奏建议

1. 先完成错误分类与 fallback-manager 接口约束，锁定状态机边界。
2. 再接入 DDG adapter 与配置层，保证主链路能跑通。
3. 随后补日志与实验埋点，避免后补时遗漏关键观测数据。
4. 最后集中完成测试、README、自测与演示准备。

## 风险与关注点

- DDG Provider 直接 import 依赖外部路径，需在技术设计阶段确认依赖稳定性与发布方式。
- 30 秒 Gemini 超时 + DDG 重试策略可能导致总耗时接近上限，需在技术设计阶段明确中止条件。
- 空响应是否视为 fallback 触发条件已经在 PRD 中定义，编码阶段需保持与错误分类规则一致。
- A/B 实验统计口径需在编码前固定，避免阶段间记录标准不一致。
