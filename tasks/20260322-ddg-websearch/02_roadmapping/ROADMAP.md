# 开发计划 - OpenClaw DDG Web Search

## 需求概述

交付一个面向 OpenClaw 内置搜索场景的全栈 Web Search 扩展，完成 DuckDuckGo 主搜索源、Bing 备用搜索源、基础反爬控制与一体化部署落地。

## 开发计划

| 任务名称 | 详细描述 | 计划工作量(人天) |
|------------|----------|------------|
| 【搜索入口契约】(后端)统一搜索请求与响应契约 | 1. 明确搜索请求入参与校验边界<br>2. 统一搜索响应、错误对象与元数据字段<br>3. 补齐健康检查与配置项定义 | 1 |
| 【主源接入】(后端)DuckDuckGo 搜索能力接入 | 1. 完成 DuckDuckGo 搜索请求与结果返回<br>2. 支持标题、链接、摘要与相关搜索提取<br>3. 覆盖主源成功与空结果场景 | 1 |
| 【备用降级】(后端)Bing 兜底搜索能力接入 | 1. 完成 Bing 备用搜索结果返回<br>2. 处理主源超时、限流、验证码与解析失败切换<br>3. 保证主备搜索源输出结构一致 | 1 |
| 【反爬控制】(后端)请求保护与失败退避 | 1. 补充浏览器级请求头与随机延迟控制<br>2. 增加超时、短时退避与可重试标记<br>3. 记录搜索源、耗时、结果数与失败原因日志 | 1 |
| 【结果消费】(前端)搜索结果接入与视图适配 | 1. 对接新扩展搜索入口并替换历史调用路径<br>2. 支持 `summary` 与 `detailed` 两种结果消费视图<br>3. 适配相关搜索、来源标记与结果不足提示 | 1 |
| 【异常反馈】(前端)降级结果与失败态展示 | 1. 处理主备切换后的返回提示与交互反馈<br>2. 展示可重试失败、空结果与部分结果场景<br>3. 保证调用侧在异常场景下可继续使用搜索能力 | 0.5 |
| 【部署发布】(后端)扩展部署与运行配置落地 | 1. 完成扩展目录交付与运行配置整理<br>2. 切换历史 `DDG Fallback` 到新扩展入口<br>3. 校验部署后健康检查与运行可用性 | 1 |
| 【全链路联调】(后端)联调测试 | 1. 验证主源成功、主备切换与全部失败场景<br>2. 验证输出契约、日志字段与健康检查结果 | 0.5 |
| 【全链路联调】(前端)联调测试 | 1. 验证结果展示、相关搜索与错误提示交互<br>2. 验证 `summary`/`detailed` 视图与降级场景消费 | 0.5 |
| 【全链路联调】演示 | 1. 准备主源成功、降级成功与失败回退演示脚本<br>2. 验证演示环境、配置项与展示链路稳定性 | 0.5 |
| **合计** | | **8** |

## 涉及模块

- `extensions/openclaw-web-search/`: 新 Web Search 扩展主体与运行配置
- OpenClaw 搜索调用入口: 搜索请求接入、历史 `DDG Fallback` 替换与结果消费
- 搜索源适配模块: DuckDuckGo 主源与 Bing 备用源能力封装
- 反爬与可观测模块: 请求头、随机延迟、退避与结构化日志
- 部署与验证链路: 扩展发布、健康检查、联调测试与演示环境
