## ISSUE-001: DDG 验证码检测不完整

**发现日期**: 2026-03-23  
**发现阶段**: 部署后验证  
**严重度**: 🔴 P0（生产环境故障）  
**状态**: ✅ 已修复

### 问题描述

**症状**:
```
【测试 1】基础搜索 (summary 模式)
✅ 搜索成功
查询：OpenClaw AI
结果数量：0  ❌（期望：5+）
搜索源：bing
```

**影响**:
- 生产环境无法获取搜索结果
- 所有使用 Web Search 功能的用户受影响

### 根因分析

**直接原因**:
1. DDG 返回新版验证码（anomaly-modal），原检测逻辑未覆盖
2. 测试使用模拟数据，未使用真实 HTML 验证
3. 缺少集成测试层级

**深层原因**:
1. 测试策略缺失"真实数据验证"环节
2. 研发流程没有定义完整的测试金字塔
3. 部署前验证流程不完善

### 修复方案

**短期（立即执行）**:
- [ ] 增强 `detectDdgBlockingState()` 检测 6 个特征
- [ ] 创建 HTML 样本库（tests/fixtures/）
- [ ] 新增集成测试验证真实样本

**中期（本周完成）**:
- [ ] 建立 HTML 样本定期更新机制
- [ ] 部署前必须运行集成测试
- [ ] 更新测试策略文档

**长期（持续优化）**:
- [ ] 建立问题驱动迭代流程
- [ ] 每次迭代更新 CHANGELOG.md
- [ ] 建立版本标签机制

### 验证结果（基线测试 - 修复前）

| 测试类型 | 用例数 | 通过数 | 失败数 | 状态 |
|---------|--------|--------|--------|------|
| 单元测试 | 10 | 10 | 0 | ✅ 通过 |
| 集成测试 | 7 | 4 | 3 | ❌ 失败 |
| 功能验证 | 7 | - | - | ⏳ 待执行 |

**失败用例分析**:
1. `验证码页面应抛出 SearchError` - 错误消息不匹配正则
2. `正常页面应提取到结果` - Bing 解析器提取 0 个结果（期望≥5）
3. `结果应包含标题、URL、摘要` - 无结果可验证

**根因确认**:
- DDG 验证码检测：`detectDdgBlockingState` 已检测到，但错误消息格式问题
- Bing 解析器：正则表达式无法匹配真实 HTML 结构（`<h2 class="">`、`<p class="b_lineclamp2">`）

### 修复结果（v1.1 - 修复后）

| 测试覆盖率 | - | 84.59% | - | ✅ 达标 |

**修复内容**:
1. `detectDdgBlockingState()` - 检测特征从 2 个扩展到 6 个
2. `extractBingResults()` - 支持灵活 class 属性匹配
3. 测试断言逻辑修复（检查 error.type 而非 error.message）
