## 1. 业务背景 (Business Context)

### 1.1 项目定位
为 OpenClaw `web_search` 工具添加 **DuckDuckGo Provider**，实现零门槛、免 API Key 的网页搜索能力。

### 1.2 商业价值
| 维度 | 现状 | 改进后 |
|------|------|--------|
| **使用门槛** | 需注册 5 个平台之一，配置 API Key | 零配置，开箱即用 |
| **成本** | $5-30/月（Brave/Perplexity 等） | 完全免费 |
| **隐私保护** | 商业服务可能有日志 | DuckDuckGo 主打隐私保护 |
| **社区吸引力** | 新手可能被 API Key 劝退 | 降低新手入门门槛 |

### 1.3 战略意义
- **降低 OpenClaw 使用门槛**：新手无需注册任何服务即可体验完整功能
- **补充现有 Provider 生态**：作为免费/隐私友好的备选方案
- **社区贡献潜力**：可提交 PR 回馈 OpenClaw 官方

## 2. 用户痛点 (User Pain Points)

### 2.1 目标用户
| 用户类型 | 特征 | 当前痛点 |
|---------|------|---------|
| **新手用户** | 首次接触 OpenClaw，想快速体验 | 被 API Key 配置流程劝退 |
| **隐私敏感用户** | 关注数据隐私，不愿注册商业服务 | 现有 Provider 都需要账号 |
| **低频用户** | 偶尔使用搜索功能，不值得付费 | Brave 等最低$5/月，性价比低 |
| **开发者** | 想扩展 OpenClaw 能力 | 缺少免 API Key 的参考实现 |

### 2.2 现有解决方案的不足
| Provider | 不足 |
|---------|------|
| **Brave** | 需注册账号、配置 API Key、有查询配额 |
| **Gemini** | 需 Google 账号、API Key、可能有数据收集 |
| **Grok** | 需 xAI 账号、API Key、付费服务 |
| **Kimi** | 需 Moonshot 账号、API Key |
| **Perplexity** | 需注册账号、API Key、有配额限制 |

## 3. 功能范围 (Feature Scope)

### 3.1 包含的功能 (In Scope) ✅

| 功能模块 | 说明 | 优先级 |
|---------|------|--------|
| **DDG HTML API 集成** | 调用 `html.duckduckgo.com/html/` 接口 | P0 |
| **HTML 解析器** | 解析搜索结果（标题、摘要、URL） | P0 |
| **参数支持** | `query`（必须）、`country`、`language` | P0 |
| **速率限制** | 自适应延迟，避免被 DDG 封禁 | P0 |
| **结果缓存** | 内存缓存 15 分钟（对齐现有 provider） | P1 |
| **错误处理** | 网络错误、解析错误、限流处理 | P0 |
| **单元测试** | 核心功能测试覆盖率 > 80% | P1 |
| **文档** | README.md + 使用示例 | P1 |

### 3.2 不包含的功能 (Out of Scope) ❌

| 功能 | 说明 | 原因 |
|------|------|------|
| **DDG Instant Answer API** | 仅支持即时答案，不支持网页搜索 | 功能不完整 |
| **DDG 图片/视频搜索** | 仅支持网页搜索 | MVP 范围控制 |
| **DDG 新闻搜索** | 仅支持通用网页搜索 | 后续迭代 |
| **高级搜索语法** | 如 `site:`、`filetype:` 等 | DDG HTML 接口不支持 |
| **分页支持** | 仅返回第一页结果（10 条） | MVP 范围控制 |
| **原生集成到 OpenClaw** | 先以 MCP Server 形式独立部署 | 降低初始风险 |

## 4. 验收标准 (Acceptance Criteria)

### 4.1 功能验收

| 编号 | 验收项 | 通过标准 |
|------|--------|---------|
| **AC-01** | 基本搜索 | 输入 `"OpenClaw"` 返回 10 条相关结果 |
| **AC-02** | 国家参数 | `country: "CN"` 返回中文优先结果 |
| **AC-03** | 语言参数 | `language: "zh-CN"` 返回中文结果 |
| **AC-04** | 结果格式 | 每条结果包含 `title`、`snippet`、`url` |
| **AC-05** | 错误处理 | 网络错误时返回友好错误信息 |
| **AC-06** | 速率限制 | 连续 10 次查询不触发 DDG 限流 |
| **AC-07** | 缓存命中 | 相同查询 15 分钟内返回缓存结果 |
| **AC-08** | 单元测试 | 所有测试用例通过，覆盖率 > 80% |

### 4.2 性能验收

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| **响应时间** | P95 < 3 秒 | 100 次查询统计 |
| **并发支持** | 支持 5 个并发请求 | 压力测试 |
| **内存占用** | < 50MB（含缓存） | 运行时监控 |
| **缓存命中率** | > 60%（典型使用场景） | 日志统计 |

### 4.3 安全验收

| 检查项 | 要求 |
|--------|------|
| **SSRF 防护** | 阻止访问内网地址 |
| **输入验证** | 查询参数长度限制（max 500 字符） |
| **URL 过滤** | 结果 URL 必须是 http/https |
| **User-Agent** | 使用标准浏览器 UA，避免被识别为爬虫 |

## 8. 附录：DDG HTML API 参考

### 8.1 接口格式
```
GET https://html.duckduckgo.com/html/
Query Parameters:
  - q (required): 搜索关键词
  - kl (optional): 国家代码，如 cn-US, de-DE
  - kd (optional): 语言，如 zh-cn, en-us
```

### 8.2 响应示例（HTML 片段）
```html
<div class="result">
  <h2 class="result__title">
    <a class="result__a" href="//...">标题</a>
  </h2>
  <a class="result__url" href="//...">域名</a>
  <div class="result__snippet">摘要...</div>
</div>
```
