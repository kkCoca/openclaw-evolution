## 🎯 产品愿景

**打造生产级搜索降级盾牌，确保 Gemini 故障时 DuckDuckGo Fallback 在 <5 秒内无缝切换，用户搜索体验零中断。**

### 核心问题陈述

当前 DuckDuckGo Fallback 实现存在以下生产环境风险：

| 问题 | 影响 | 严重度 |
|------|------|--------|
| 决策超时配置 (4.5 秒) 接近临界值 | 高负载时可能超时失效 | 🔴 高 |
| 缺少端到端延迟监控 | 无法验证<5 秒 SLA | 🟡 中 |
| 熔断恢复策略过于简单 | 可能导致频繁切换 | 🟡 中 |
| 缺少用户感知优化 | 切换过程可能有卡顿 | 🟡 中 |

---

## 📋 需求定义

### 功能需求 (FR)

#### FR-001: <5 秒无缝切换 SLA
**描述**: 当 Gemini 搜索触发 Fallback 时，从故障检测到 DuckDuckGo 返回结果的总延迟必须 <5 秒。

**验收标准**:
- [ ] 决策超时 (fallbackDecisionTimeoutMs) ≤ 4000ms (预留 1 秒缓冲)
- [ ] DDG 适配器超时 (ddgTimeoutMs) ≤ 8000ms (含重试)
- [ ] 端到端延迟 P95 < 5000ms (生产环境监控)
- [ ] 用户无感知切换 (无错误提示、无加载中状态重置)

#### FR-002: 智能熔断与健康检查
**描述**: 基于 Gemini 健康状态的动态熔断机制，避免无效重试。

**验收标准**:
- [ ] 连续失败 3 次触发熔断 (failureThreshold=3)
- [ ] 熔断持续时间 5 分钟 (circuitBreakerTimeoutMs=300000)
- [ ] 恢复阈值：连续成功 2 次 (recoveryThreshold=2)
- [ ] 熔断状态持久化 (跨会话保持)

#### FR-003: 多级降级策略
**描述**: 根据错误类型智能选择降级策略，而非一刀切。

**验收标准**:
- [ ] 429/503/Timeout → 立即触发 Fallback (无需等待决策超时)
- [ ] 空响应 → 触发 Fallback (可配置)
- [ ] 认证错误/权限错误 → 直接报错 (不触发 Fallback)
- [ ] 网络错误 → 重试 1 次后触发 Fallback

#### FR-004: 用户感知优化
**描述**: 切换过程中保持用户界面稳定，避免体验中断。

**验收标准**:
- [ ] 切换时不重置搜索框内容
- [ ] 切换时不刷新页面/组件
- [ ] 可选：显示"正在切换备用搜索引擎..."提示 (可配置)
- [ ] 切换后保留搜索历史上下文

#### FR-005: 可观测性与监控
**描述**: 完整的日志与指标，支持生产环境监控。

**验收标准**:
- [ ] 结构化日志 (requestId, queryHash, elapsedMs, fallbackUsed)
- [ ] 关键指标：Fallback 触发率、平均切换延迟、DDG 成功率
- [ ] 健康状态端点 (GET /health)
- [ ] 告警规则：Fallback 触发率 > 10% 持续 5 分钟

---

### 非功能需求 (NFR)

#### NFR-001: 性能
- P95 端到端延迟 < 5000ms (含 Fallback)
- P99 端到端延迟 < 8000ms
- 吞吐量 ≥ 100 请求/秒

#### NFR-002: 可靠性
- Fallback 触发成功率 ≥ 99.9%
- DDG 搜索成功率 ≥ 95% (生产环境)
- 熔断误触发率 < 1%

#### NFR-003: 可维护性
- 单元测试覆盖率 ≥ 90%
- 集成测试覆盖所有错误场景
- 文档完整度 100% (PRD/ROADMAP/SPEC/README)

#### NFR-004: 安全性
- 不记录用户搜索内容 (仅记录 queryHash)
- 不泄露 API Key 或认证信息
- 支持代理配置 (HTTP_PROXY/HTTPS_PROXY)

## 🔄 逻辑闭环设计

### 状态机流转图

```
┌─────────────┐
│   Healthy   │ ◄────────────────────┐
│  (Gemini OK)│                      │
└──────┬──────┘                      │
       │ 429/503/Timeout             │ 连续成功 2 次
       ▼                              │
┌─────────────┐                      │
│   Degraded  │──────────────────────┘
│ (Fallback)  │
└──────┬──────┘
       │ 连续失败 3 次
       ▼
┌─────────────┐
│  Unhealthy  │ ──┐
│  (Circuit   │   │ 5 分钟后
│   Open)     │ ──┘
└─────────────┘
```

### <5 秒切换时序图

```
用户搜索请求
    │
    ▼
┌───────────────────────┐
│  Gemini 搜索 (0-4s)   │
└───────────┬───────────┘
            │ 429/503/Timeout
            ▼ (t=4.0s)
┌───────────────────────┐
│  错误分类 (<100ms)    │
└───────────┬───────────┘
            │ 触发 Fallback
            ▼ (t=4.1s)
┌───────────────────────┐
│  DDG 搜索 (4.1-4.8s)  │
└───────────┬───────────┘
            │ 返回结果
            ▼ (t=4.8s)
┌───────────────────────┐
│  用户收到结果 (<5s)   │
└───────────────────────┘
```

### 关键路径延迟预算

| 阶段 | 预算 | 说明 |
|------|------|------|
| Gemini 搜索 | 0-4000ms | 决策超时阈值 |
| 错误分类 | <100ms | 本地同步操作 |
| Fallback 决策 | <100ms | 状态检查 + 配置读取 |
| DDG 搜索 | 4000-4800ms | 含重试 (最多 3 次) |
| 结果返回 | <200ms | 序列化 + 响应 |
| **总计** | **<5000ms** | **SLA 目标** |

## 📊 成功指标

### 核心指标 (North Star Metrics)

| 指标 | 目标值 | 测量方式 |
|------|-------|---------|
| **Fallback 触发率** | < 5% | (Fallback 次数 / 总搜索次数) × 100% |
| **切换延迟 P95** | < 5000ms | 从 Gemini 故障到 DDG 返回的延迟 |
| **DDG 成功率** | ≥ 95% | (DDG 成功次数 / Fallback 触发次数) × 100% |
| **用户满意度** | ≥ 4.5/5 | Moltbook 反馈 + 社区调研 |

### 辅助指标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 熔断误触发率 | < 1% | Gemini 实际可用但触发熔断的比例 |
| 平均修复时间 (MTTR) | < 10 分钟 | 从故障检测到恢复的时间 |
| 日志完整率 | 100% | 所有搜索请求都有结构化日志 |

## 🚨 异常场景处理

### 场景 1: Gemini 429 限流
**触发条件**: HTTP 429 Too Many Requests  
**响应策略**: 
1. 立即触发 Fallback (跳过决策超时)
2. 记录 Gemini 健康状态 (consecutiveFailures+1)
3. 如果连续失败≥3 次，触发熔断

### 场景 2: DDG 搜索也失败
**触发条件**: DDG 重试 3 次后仍失败  
**响应策略**:
1. 抛出 SmartSearchError (包含 Gemini 和 DDG 错误信息)
2. 记录完整错误链 (用于调试)
3. 不触发进一步降级 (避免无限递归)

### 场景 3: 熔断期间用户搜索
**触发条件**: Gemini 状态=unhealthy 且未到恢复时间  
**响应策略**:
1. 直接走 DDG Fallback (跳过 Gemini 尝试)
2. 记录"熔断跳过"日志
3. 检查是否达到恢复时间，如达到则尝试 Gemini

### 场景 4: 网络完全不可用
**触发条件**: 无网络连接或代理配置错误  
**响应策略**:
1. Gemini 和 DDG 均失败
2. 抛出明确的网络错误 (非通用错误)
3. 建议用户检查网络连接
