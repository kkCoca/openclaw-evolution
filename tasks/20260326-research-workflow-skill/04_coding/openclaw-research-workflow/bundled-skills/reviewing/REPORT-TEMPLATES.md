# 报告模板

## 审查报告

```markdown
# 审查报告

## 范围

- **代码**：[文件列表]
- **需求文档**：[文档列表，无则写"无"]
- **技术文档**：[文档列表，无则写"无"]

## 需求审查

[有需求文档时输出，无则删除此节]

- [ ] 功能必要性：[通过/问题]
- [ ] 需求自洽：[通过/问题]
- [ ] 边界覆盖：[通过/问题]
- [ ] 文档一致性：[通过/问题]

## 架构审查

[有技术文档时输出，无则删除此节]

- [ ] 方案简洁性：[通过/问题]
- [ ] 变更影响：[通过/问题]
- [ ] 职责清晰：[通过/问题]
- [ ] 文档一致性：[通过/问题]

## 代码审查

### 摘要

| 级别 | 数量 |
|------|------|
| 🚫 阻塞 | N |
| ⚠️ 必报 | N |
| 💡 建议 | N |

### 问题列表

[按严重程度排序，阻塞优先]

## 结论

**[通过 / 不通过]**

[原因说明]
```

## 问题格式

```markdown
### [🚫/⚠️/💡] 问题 N：[标题]

**位置**：`文件:行号`

**问题**：[描述 + 为什么是问题]

**建议**：[修复方案，复杂问题可列多个选项]
```

## 多选项格式（复杂问题）

```markdown
### [🚫/⚠️/💡] 问题 N：[标题]

**位置**：`文件:行号`

**问题**：[描述]

**选项**：

| 选项 | 优点 | 缺点 |
|------|------|------|
| A [推荐] | ... | ... |
| B | ... | ... |
```
