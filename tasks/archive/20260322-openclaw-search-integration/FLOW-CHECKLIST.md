## 🚦 流程控制规则 (单步验证机制)

### 核心原则

**每个步骤执行完后立即验证，验证通过后才允许进入下一步**。

```
步骤 N 执行完成 → OpenCode 自检 → @OpenClaw 验证 → 验证通过 → 步骤 N+1 开始
                                          ↓
                                      验证不通过 → 修复 → 重新验证
```

### 验证响应时间

| 验证类型 | 响应时间 | 说明 |
|---------|---------|------|
| OpenCode 自检 | 即时 | 步骤完成后立即自检 |
| OpenClaw 验证 | <5 分钟 | OpenCode 完成后@OpenClaw，5 分钟内响应 |
| 验证不通过 | 立即退回 | 发现问题立即退回修复 |

### 验证通过标准

| 步骤 | 验证通过标准 |
|------|-------------|
| designing | PRD.md + TRD.md 文件存在 + 内容符合模板 |
| roadmapping | ROADMAP.md 文件存在 + 包含任务表格 + 联调测试项 + 演示项 |
| detailing | DETAIL.md 文件存在 + 按文件组织 + 方法签名完整 |
| coding | npm test 通过 + npm run build 通过 |
| reviewing | REVIEW_REPORT.md 结论为"通过" |
| L2/L3/L4 | 对应检查脚本输出"✅ 通过" |

### 异常处理

| 异常 | 处理方式 |
|------|---------|
| 步骤产出物缺失 | 立即退回该步骤，补充产出物 |
| 步骤验证不通过 | 立即修复问题，重新验证 |
| OpenCode 跳过步骤 | OpenClaw 立即提醒，要求退回补充 |
| reviewing 不通过 | 修复问题，重新 reviewing |
| OpenClaw 验证超时 (>5 分钟) | OpenCode 主动@OpenClaw 提醒 |
| OpenClaw 越权行动 | 主人立即纠正，撤销修改，写入 AGENTS.md |
