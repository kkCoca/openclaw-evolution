## 🔄 核心流程

### 正常流程

```
1. OpenClaw 内部搜索模块发起搜索请求
   ↓
2. Web Search 扩展校验输入参数，生成请求上下文
   ↓
3. 按反爬策略设置请求头与随机延迟，优先请求 DuckDuckGo
   ↓
4. 解析 DuckDuckGo 页面结果，提取标题、链接、摘要和相关搜索词
   ↓
5. 按输出格式组装响应，返回给 OpenClaw 调用方
   ↓
6. 记录成功日志（搜索源、耗时、结果数量）
```

### 失败处理流程

```
查询词为空 → 直接返回参数错误，不发起外部请求
DuckDuckGo 超时 → 记录超时原因并自动切换到 Bing
DuckDuckGo 返回限流/验证码 → 判定为主源不可用，进入备用搜索源
DuckDuckGo 解析失败 → 记录解析失败并切换到 Bing
Bing 也失败 → 返回明确失败结果（包含失败类型与可重试标记）
返回结果少于 10 条 → 只返回实际结果，元数据中标记结果不足
相关搜索缺失 → 主结果照常返回，相关搜索字段为空数组
```

## 📋 范围定义

### 范围内（In Scope）

- ✅ 面向 OpenClaw 的 Web Search 扩展设计与交付
- ✅ DuckDuckGo 主搜索源接入
- ✅ Bing 免 API 备用搜索源接入
- ✅ 基础搜索结果解析
- ✅ 相关搜索词提取
- ✅ 输出格式配置（summary/detailed）
- ✅ 基础反爬与失败降级
- ✅ 替换历史 `DDG Fallback` 的集成入口

### 范围外（Out of Scope）

- ❌ 代理池、IP 轮换、验证码自动识别
- ❌ 多区域搜索策略和个性化排序
- ❌ 搜索结果缓存平台或持久化数据仓库
- ❌ 图片搜索、新闻搜索、学术搜索等垂类能力
- ❌ 面向第三方开发者的公网 API 服务

## 🔗 边界与约束

| 边界 | 说明 |
|------|------|
| **使用场景** | 仅服务于 OpenClaw 内置搜索，不提供公共开放 API |
| **搜索范围** | 仅支持通用 Web Search，不支持图片、视频、新闻、地图等垂类搜索 |
| **输入语言** | 默认支持中英文查询；其他语言不保证相关搜索质量 |
| **输出条数** | 默认 10 条，最大 20 条，避免高频大结果集触发限流 |
| **反爬策略** | 本期仅实现请求头伪装与随机延迟，不引入代理池、浏览器自动化或账号体系 |
| **数据持久化** | 本期不建设长期索引库，不保存搜索结果历史 |
| **兼容关系** | 新扩展发布后，历史 `DDG Fallback` 仅保留归档，不再作为运行依赖 |
